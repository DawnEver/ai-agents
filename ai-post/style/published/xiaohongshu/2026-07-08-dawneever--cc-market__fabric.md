# 指挥一支会记事的 AI 团队

🔥 平常写大型项目时候，我很想同时开三个 AI：Claude 定方案，复制到 DeepSeek 让它写，再贴回 Codex 让它挑刺。三个窗口来回横跳，复制粘贴复制粘贴，手都抽筋。

🛠️ 我受不了了，自己写了个东西叫 fabric——一个 Claude Code 插件。装上以后我不用离开手头的终端，就能把活儿派给任何一个模型。

💡 说白了就是，我在一个终端里当导演，指挥一支模型团队。走量的粗活丢给 DeepSeek，便宜、能扛量；审查和出图交给 Codex（它带原生生图）；Claude（顶尖模型）在最上面总控，拆任务、分活儿。谁适合干啥就干啥，我再也不用在开工前纠结"这个子任务到底该喊哪个模型"。

⚡ 但真正让我上头的，是它们现在会记事了。

💬 以前派活儿是一锤子买卖：我把上下文一股脑塞过去，模型答完就忘，下一句又是白纸，像跟一个每天失忆的实习生共事，每次都得从头讲。现在换成了能持续来回的会话——我随口塞给 DeepSeek 一个暗号，聊到第五六轮回头再问，它居然还记得。交接从一次性，变成了真能一起把事情做完的协作。

⭐ 这东西其实是我之前的插件 takeover 一路长出来的。takeover 一代就一段脚本、二代成了 MCP server，可它只会一次性派活、模型之间没记忆。我想让几个模型真能一起干活，一路加会话、加记忆、把好几家模型拧到一起，长着长着就成了 fabric——名字就是"把工作流编织到一起"的意思。

✨ 现在它有 8 个工具、5 种模式，claude、codex、deepseek 三家随便调，而且整个引擎零依赖，纯 node 内置库，装起来干干净净。

🔬 而且它能干的远不止派活——我最近还拿 fabric 搭了个台子,去扒 Claude Code 的缓存黑箱,那篇讲了更进阶的玩法(切 effort 到底动不动缓存)。想看硬核的翻我主页。

👇 一行装：/plugin install fabric@cc-market。GitHub 搜 DawnEver/cc-market，fabric 就在里面。想要直链评论区戳我～
