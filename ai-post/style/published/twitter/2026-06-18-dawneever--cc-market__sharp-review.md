# sharp-review: let DeepSeek, Claude & Codex each critique your code from their own angle

**Tweet 1 (Hook):**
A model reviewing its own code isn't a review. It can't catch what it couldn't catch the first time — same blind spots, same misses.

The fix isn't a smarter model. It's a rival one. 🧵👇

<!-- 192 chars -->

**Tweet 2 (Feature):**
So I built sharp-review: DeepSeek, Claude and Codex each critique the same code independently. Running 5 copies of one model is parallel, not independent — they share its failure modes. I rotate 2 of 3 rival vendors per run, each from its own angle.
<!-- 247 chars -->

**Tweet 3 (Feature):**
Disagreement is the signal. Council plugins debate until models agree — which quietly buries a true minority finding. I keep the divergence. Two reviewers hitting the same thing = high confidence. A lone flag still gets surfaced, not dropped.
<!-- 238 chars -->

**Tweet 4 (Aha / How it works):**
And it doesn't nag. Most bots treat every save like a 3am page. A wave gate accumulates change across sessions and skips lockfiles, generated and doc-only diffs — so it fires on real work. 84% of devs use AI review; only ~1/3 trust it. Noise is why.
<!-- 246 chars -->

**Tweet 5 (Context):**
It's local and in-loop — not a SaaS PR bot. The same engine reviews code or prose, and every finding lands in my task list automatically instead of scrolling off in chat.
<!-- 169 chars -->
