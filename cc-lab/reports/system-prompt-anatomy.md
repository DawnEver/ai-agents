# System Prompt Anatomy (Claude Code v2.1.199)

Captured 2026-07-03 via CCDS reverse proxy → Anthropic API (`claude-haiku-4-5-20251001`).
Source: `claude-tap` trace session `08c13b96`.

---

## Overview

The system prompt has **3 text blocks**, totaling **~15.9 KB** (15,893 chars). Blocks 1 and 2 carry
`cache_control: { type: "ephemeral", ttl: "1h" }` — the server may cache them for one hour, but
the cache is effectively orphaned per session because Block 2 embeds two session-unique paths
(cwd + scratchpad UUID). This matches the finding in `reports/prompt-anatomy.md` (M4).

| Block | Size | Cached | Purpose |
|-------|------|--------|---------|
| 0 | 70 B | No | Billing/telemetry header |
| 1 | 57 B | Yes | Identity one-liner |
| 2 | 15,766 B | Yes | All instructions, environment, git status |

Block 0 is always fresh (no `cache_control`) — it carries the CC version string used for
billing attribution on every request.

## Block 0 — Billing Header

```
x-anthropic-billing-header: cc_version=2.1.199.ab9; cc_entrypoint=cli;
```

A custom HTTP header injected as a system-prompt text block (not a real HTTP header). Tells the
API backend which client and version is making the request. Changes with every CC release.

## Block 1 — Identity

```
You are Claude Code, Anthropic's official CLI for Claude.
```

The shortest block. Always cached. Establishes the agent's identity.

## Block 2 — Full Instruction Block

### Static sections (same every session)

| Section | Content |
|---------|---------|
| **Role & scope** | "interactive agent that helps users with software engineering tasks" |
| **Security boundaries** | Auth testing ok, DoS/malware not ok; dual-use tools need context |
| **URL policy** | Never generate/guess URLs; may use user-provided URLs |
| **System rules** | Tool execution, permission modes, hooks, context compression |
| **Doing tasks** | Code style (minimal, no premature abstraction), editing over creating, security awareness |
| **Executing actions** | Risk assessment framework, confirmation rules, examples of destructive ops |
| **Using tools** | Prefer dedicated tools over Bash, call tools in parallel, TaskCreate workflow |
| **Tone & style** | Short, concise, no emojis unless asked, `file_path:line_number` references |
| **Text output rules** | One-sentence updates, no narration, no multi-paragraph docstrings |
| **Tool definitions** | 35 tools (Read, Write, Edit, Bash, Glob, Grep, Agent, TaskCreate, etc.) |

### Dynamic sections (change every session)

These are the parts that make the cache "orphaned" — the server sees a different block every
session even though 95% of the text is identical:

1. **Model identity paragraph** — `"You are powered by the model named Haiku 4.5. The exact model ID is claude-haiku-4-5-20251001."` Changes per model selection.

2. **Environment block** — Primary working directory, git repo flag, platform, shell, OS version. The cwd path is unique per session (each run gets a timestamped `.lab/` dir).

3. **Scratchpad directory** — A UUID-based temp path unique to each session.

4. **gitStatus block** — Snapshot of git status at session start (branch, dirty files, recent commits).

These four dynamic sections are why the prompt cache doesn't survive across sessions even
though the block carries `cache_control: ephemeral`.

## Cross-Backend Comparison

### What's the same

The system prompt is **generated by the Claude Code client** before it sends the API request.
The client code is the same regardless of which backend you point it at. So these are identical
across Anthropic, DeepSeek, or any other provider:

- All 3 blocks (billing header, identity, instruction block)
- All 35 tool definitions
- All static instruction text
- The `CLAUDE.md` / `AGENT.md` user context (injected into messages, not system prompt)

### What differs

| Dimension | Anthropic (direct) | DeepSeek (via CCDS) |
|-----------|-------------------|---------------------|
| `model` field | `claude-haiku-4-5-20251001` | `deepseek-v4-pro[1m]` (or similar) |
| `thinking` block | `{ type: "enabled", budget_tokens: 31999 }` | May be absent or ignored if unsupported |
| Model identity line | "Haiku 4.5" / "Sonnet 5" / etc. | Whatever CCDS reports as the active model |
| Actual behavior | Native Anthropic tool use | Model-dependent — same instructions, different execution |

### Key insight

The system prompt doesn't know or care about the backend. It always says "You are Claude Code,
Anthropic's official CLI for Claude" even when routing to DeepSeek. The model receives the
same identity, the same rules, and the same tools. **Differences in output quality or tool-use
accuracy come from the model, not the prompt.**
